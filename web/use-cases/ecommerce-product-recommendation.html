E-commerce Hybrid Product Recommendation Pipeline: Personalised at Scale

Combines collaborative filtering, content-based signals, and ensemble learning into a single hybrid engine — serving personalised product listings with a continuous feedback loop and real-time A/B testing.

The Recommendation Problem

Basic recommendation engines rely on a single signal — either what the customer bought before, or what's similar to the current product. Both approaches miss a large share of relevant items. A hybrid engine combines user behaviour, collaborative patterns, and item content to produce a ranked list that's both personalised and contextually relevant, then keeps improving itself through every click and conversion.

Before the Hybrid Pipeline

Single-signal recommendations miss items that don't fit the narrow filter

New users get generic or popularity-only recommendations — no personalisation

No feedback loop — click and conversion data isn't fed back to improve future results

No A/B testing infrastructure — impossible to measure which recommendation strategy converts better

How the Pipeline Works

1

Data Sources Ingested

User Behaviour (browsing history, click events, cart additions, purchases), Collaborative Data (co-occurrence statistics), and Content Data (item features, metadata, embeddings) are collected continuously

2

Collaborative Model

Matrix Factorization and Nearest Neighbours models generate an initial candidate set based on what similar users have viewed and purchased

3

Candidate Generation

The candidate pool is filtered and narrowed before entering the main ranking stage — removing irrelevant or out-of-stock items

4

Hybrid Recommendation Engine

Ensemble Learning combines collaborative and content signals; Product Embeddings capture semantic item relationships; Re-rank Candidates produces the final personalised order

5

Online Serving

Recommended Listings and Personalisation are served to the shopper; Real-time A/B Testing runs simultaneously to compare recommendation strategies

6

Feedback Loop

Click and Conversion events are fed back into the pipeline — the model improves continuously with each shopper interaction

Technical Stack

Matrix Factorization & Nearest Neighbours

Collaborative filtering models that identify patterns across user behaviour

Product Embeddings

Encodes item features, metadata, and semantic relationships into vector space

Ensemble Learning

Combines multiple model signals into a single, stronger ranking output

Real-time A/B Testing

Continuously measures which recommendation strategy drives higher conversions

What Changes

Hybrid

Three signal types combined — collaborative, content, and behavioural

Real-time

Recommendations served live with A/B testing running continuously

Self-improving

Feedback loop feeds every click and conversion back into the model

Personalised

Each shopper receives a ranked list specific to their behaviour and preferences

Systems This Connects To

E-commerce Platform (Shopify, Magento, custom) User Behaviour Tracking (clickstream) Product Catalogue / Inventory Vector Database (embeddings store) A/B Testing Layer Analytics / Conversion Tracking
